## Project Overview

**Yu-Gi-Oh Card Scanner** — Android app built in Kotlin that uses the camera and ML Kit OCR to scan and identify Yu-Gi-Oh cards, with local persistence and remote API integration.

**Stack:** Kotlin 2.1.0, AGP 9.2.1, Jetpack Compose (BOM 2026.02.01), Room 2.7.0, CameraX 1.3.4, ML Kit Text Recognition 16.0.1, Gson 2.11.0, Coil 2.7.0, Firebase (Auth + Firestore), KSP 2.1.0-1.0.29

> **Arquitectura offline-first (rediseño 2026-06-14).** La app NO depende de ningún backend
> en runtime: busca cartas en un catálogo local (Room) empaquetado en `assets/`. Firebase
> solo se usa para login (Google) y backup/restore opcional. Retrofit/PostgreSQL se
> eliminaron. Ver `docs/ARQUITECTURA.md` y `docs/ROADMAP.md` para el detalle y el estado por
> fases.

**Target:** SDK 35, min SDK 26, Java 11

## Architecture

**MVVM + Clean Architecture:**
- **Data layer** (api/, db/, model/): Retrofit + Room for persistence, API response models
- **UI layer** (ui/screens/, ui/viewmodel/): Compose UI, ViewModels manage state via StateFlow/MutableStateFlow
- **Separation:** UI doesn't access database or API directly; flows through ViewModel
- **State management:** `EstadoBusqueda` sealed class in ScannerViewModel models async states (Inactivo, Cargando, Exito, Error)

**Key flows:**
1. User searches card → ViewModel calls Retrofit → Updates StateFlow → UI recomposes
2. Scanned text → Attempts translation via TraduccionInstance API → Falls back to original text
3. Results saved → Room database insertion via CartaDao

## Key Configuration Notes

**Kotlin 2.1.0 + AGP 9.2.1:**
- `android.disallowKotlinSourceSets=false` in gradle.properties required for KSP 2.1.x + AGP 9.x compatibility
- Configuration cache enabled (`org.gradle.configuration-cache=true`)
- Do NOT declare `kotlin-android` plugin explicitly; AGP 9.x handles it
- Do NOT add `kotlinOptions` blocks; Kotlin plugin configuration is implicit

**Version Catalog:**
All dependencies managed in `gradle/libs.versions.toml`. When adding or updating dependencies:
- Update version in `[versions]` section
- Add library alias in `[libraries]` section
- Reference with `libs.` prefix in build.gradle.kts (e.g., `libs.retrofit`)
- Never hardcode versions in build files

**KSP (Kotlin Symbol Processing):**
- Room uses KSP for code generation (@Database, @Entity, @Dao)
- KSP configuration in app/build.gradle.kts: `ksp(libs.room.compiler)`
- Generated Room code goes to `build/generated/ksp/debug/` after compilation

## Development Patterns

**ViewModels:**
- Inherit from `androidx.lifecycle.ViewModel`
- Use `viewModelScope` for coroutine management (auto-cancelled on ViewModel destruction)
- Expose StateFlow for UI to observe: `val estado: StateFlow<EstadoBusqueda>`
- Keep mutable state private: `private val _estado = MutableStateFlow(...)`

**Compose Screens:**
- Simple state management with `remember { mutableStateOf(...) }` for local UI state
- Collect StateFlow in Compose with `.collectAsState()` for ViewModel state
- Navigation between screens managed via sealed class (e.g., `AppScreen.Scanner`)

**Database Access:**
- Room DAOs are abstract interfaces; implementation generated by KSP at compile time
- Suspend functions for database operations (e.g., `suspend fun insertCarta(...)`)
- Database is singleton per app (see `AppDatabase.getInstance()`)

**Retrofit:**
- Service interfaces define endpoints with `@GET`, `@Query`, etc.
- Response models use data classes with `@SerializedName` for JSON field mapping
- RetrofitInstance and TraduccionInstance handle API initialization

## Project-Specific Guidance

**Camera & CameraX:**
- CameraScreen uses CameraX for preview and frame analysis
- ML Kit OCR processes frames to extract text
- Permissions required: CAMERA, INTERNET (declared in AndroidManifest.xml)

**API Integration:**
- Primary: YuGiOh API (cardinfo.php endpoint, searches by card name)
- Translation: MyMemory API (translates scanned Spanish text to English for search)
- Both wrapped in Singleton instances for reuse

**Styling:**
- Theme in ui/theme/ (Material 3, Colors, Typography)
- Applied globally in MainActivity via YuGiOhScannerTheme Composable
